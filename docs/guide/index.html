Overview

This project builds and publishes Dota 2 analytics reports (monthly, patch-based, user-specific, leagues) to GitHub Pages under the docs/ folder.

60‑second summary

  • You open a report page in your browser.
  • The page reads small JSON files we already prepared.
  • You sort tables, switch tabs, and share a link to the exact view.

What you get

  • Fast, static pages. No logins. Works on mobile.
  • Players, heroes, wards, and draft insights.
  • League wrappers and monthly overviews.

Where the data comes from

  • Public OpenDota APIs
  • Light caching in this repository
  • Pre‑aggregations for speed

Key features

  • Static, fast reports hosted on GitHub Pages (no server required).
  • Automated fetching from OpenDota and report generation via PowerShell scripts.
  • Rich viewers: draft, players, heroes, teams, lane duos, ward map, league wrappers.
  • Robust keyboard accessibility and screen reader support across the site.

How this site is organized

  • docs/ – Published site (reports and this documentation).
  • data/ – JSON inputs and caches.
  • scripts/ – PowerShell automation for ingest and generation.

Friendly data flow

New here? Start with Getting started for a 2‑minute tour.
GitHub Pages: Settings → Pages → Deploy from branch → Branch: main, Folder: /docs. Your documentation is reachable at docs/guide/ and reports at docs/.

Quick start

  1. Install prerequisites: PowerShell 7+ recommended; Git; optional Python for local tooling.
  2. Configure STEAM_API_KEY in repository Secrets if CI is used.
  3. Run data fetch and generate dynamic wrappers locally (see Development page).
  4. Commit output under docs/; GitHub Pages serves it automatically.
