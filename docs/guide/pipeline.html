Data pipeline

In simple terms: we fetch public data, tidy it up, prepare small files, and publish fast web pages.

At a glance

1) Fetch

Download match and constants from OpenDota into data/cache/.

2) Prepare

Create compact JSONs (data/matches/YYYY-MM.json) and constants for the site.

3) Publish

Generate HTML wrappers under docs/ and open them in your browser.

Scripts

  • scripts/fetch_opendota_data.ps1 – Fetch constants, matches, manifests; sanitize and cache.
  • scripts/create_dynamic_reports.ps1 – Build monthly/patch wrapper pages, update docs/reports.json.
  • scripts/create_league_report.ps1 – Build a league wrapper (e.g., The International).
  • scripts/create_user_report.ps1 – Temporary user-focused wrappers and cleanup.

Folders

  • data/ – Input JSONs (heroes, maps, manifests) and cache (cache/OpenDota/).
  • docs/ – Published site with index.html, dynamic.html, league pages, images, and JS viewers.

Schedule (optional)

  • Daily: refresh constants (heroes/patch) and new matches.
  • Monthly: rebuild the “last 30 days” report and update reports.json.

Local run

  1. Open a PowerShell prompt at the repo root.
  2. Fetch data: run scripts/fetch_opendota_data.ps1.
  3. Generate pages: run scripts/create_dynamic_reports.ps1.
  4. Open docs/index.html to browse.
