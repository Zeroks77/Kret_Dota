Development

Where to click on the site
  • Start at docs/index.html for the list of reports.
  • Open a month or league, then use the tabs to switch viewers.
  • Use “Copy link” to share exactly what you’re seeing.

Prerequisites

  • Windows: PowerShell 7+ recommended.
  • Git installed and configured.
  • Optional: Python or Node.js for personal tooling; not required by the site.

Local workflow

  1. Fetch or update data with scripts/fetch_opendota_data.ps1.
  2. Generate wrapper pages with scripts/create_dynamic_reports.ps1.
  3. Open docs/index.html in a browser and test viewers.

GitHub Pages

  1. Settings → Pages → Deploy from branch → Branch main, folder /docs.
  2. Commit generated files under docs/. Pages publishes automatically.
  3. Documentation lives under docs/guide/.

Project structure

  • docs/ – Published site, reports, static assets, and docs.
  • data/ – Input data and caches.
  • scripts/ – PowerShell scripts to fetch and build.
