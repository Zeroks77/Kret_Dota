Accessibility

We aim for an inclusive experience with good keyboard and screen reader support across all pages.

What this means for you
  • Everything works with keyboard only.
  • Screen readers hear clear names and live updates.
  • Links and dialogs return focus where you expect.
  • Tables announce sorting so you’re never lost.

Implemented

  • ARIA tabs on main navigation; scoped arrow key handling.
  • Loading indicators with text and hidden live regions where appropriate.
  • Tables: captions, aria-sort on headers, keyboard sorting.
  • Focus visibility that respects high-contrast and prefers-reduced-motion.
  • Ward viewer: focusable map spots, keyboardable team/player lists and pins, flyout has role="dialog", body is aria-live, and focus returns to opener on close.
  • Segmented controls across viewers support Enter/Space and reflect state via aria-pressed.
  • Clipboard actions announce via a polite live region in Ward viewer.

Keyboard quick reference

  • 1–0: main tab shortcuts.
  • ←/→: move in a tablist; Home/End jump; Enter/Space activate.
  • Wards: Enter/Space opens details; Esc closes and returns focus.

Open items

  • Optional: promote Draft primary segmented control to ARIA tabs.
